Prevención de ataque Cross Site Scripting (XSS)

A continuación se explicará como se llevó acabo la prevención de ataques cross site.

Validación de datos

Como se conoce, los ataques XSS basan su poder en las pocas validaciones en las entradas de los usuarios, permitiendo mostrar datos que ellos mismos puedan ingresar sin validación alguna, esto les permite inyectar código malicioso que puede perjudicar a otros usuarios o al servidor.

Para evitar esto, se implementó una función llamada validacion_de_datos($input) en la cual se evaluarán las entradas de los usuarios mediante diferentes funciones, para eliminar de la misma, los posibles caracteres especiales. Entre estas funciones tenemos:

  • trim(), para quitar los espacios.
  • stripslashes(), para quitar los backslashes.
  • strip_tags(), para quitar cualquier etiqueta HTML.
  • htmlspecialchars(), para quitar los caracteres especiales.

Retorna:

  • Un string sin ningún tipo de carácter especial.
